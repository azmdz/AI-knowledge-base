# 社内ナレッジベース — 使用ガイド

## 概要

**行動指針：Done is better than perfect**

「書き殴った初稿はあるが、クオリティが不安で公開できていない」という状態を解消する機能にフォーカスしたサービスです。AI が社内文書の水準に合わせて整形・校正するため、ユーザーは内容の事実確認だけ行えば公開できます。品質判断はサービスが担います。

汎用 AI（ChatGPT 等）との違いは、社内向けに調整されたプロンプトと社内記事データを使って校正する点です。「これでいいか？」を自分で判断する手間が省け、公開へのハードルが実質的に下がります。

公開が増えると検索の質が上がり、検索で答えが見つかる、利用してもらうとサービス内での成功体験が生まれ、また記事を書きたくなる正のループへ転換することを設計上の目標としています。

---

## MVP 機能仕様

| ID | 機能 | 説明 |
|----|------|------|
| FR-1 | 記事一覧 | 公開記事の一覧表示。タグフィルタ・新着順/人気順切り替え |
| FR-2 | 記事詳細 | Markdown レンダリング、閲覧数自動インクリメント、リアクション |
| FR-3 | 記事投稿・編集 | Markdown エディタ、AI 整形・タグ提案、AI 編集指示、下書き保存 |
| FR-4 | AI Q&A 検索 | 自然言語での質問に RAG で回答生成。引用記事リンク付き |
| FR-5 | 下書き管理 | 自分の下書き一覧・編集への導線 |

### MVP スコープ外（意図的に除外）

| 機能 | 理由 |
|------|------|
| 認証・ログイン | ユーザー切り替え UI で代替（社内 MVP での簡略化） |
| ページネーション | シードデータ規模では不要 |
| Slack 通知連携 | 構想段階 |
| 学習データの蓄積・活用 | 構想段階 |
| リアクション集計ダッシュボード | データ蓄積後に検討 |

## セットアップ

```bash
npm install
cp .env.example .env       # 環境変数を設定
npx prisma migrate deploy  # マイグレーション実行
npx prisma db seed         # シードデータ投入（記事・ユーザー）
npm run dev                # 開発サーバー起動（http://localhost:3000）
```

### 環境変数

| 変数 | 必須 | 説明 |
|------|------|------|
| `DATABASE_URL` | ✓ | SQLite ファイルパス（`file:./dev.db`） |
| `AI_PROVIDER` | ✓ | `mock` / `openai` / `anthropic` |
| `OPENAI_API_KEY` | openai 時 | OpenAI API キー |
| `ANTHROPIC_API_KEY` | anthropic 時 | Anthropic API キー |
| `ANTHROPIC_MODEL` | − | Claude モデル名（省略時: `claude-opus-4-8`） |

`AI_PROVIDER=mock` ではキーなしで動作します（整形は簡易変換、検索は TF-IDF キーワードマッチ）。

## 画面ごとの操作方法

### ユーザー切り替え（共通）

画面右上のユーザーセレクターで操作ユーザーを切り替えます。選択したユーザーが記事の投稿者・編集者として扱われます（localStorage に保持）。  
MVP段階では認証は用意せず、選択形式での切替を採用しております。

---

### 記事一覧（`/`）

- **並び替え**: 右上の「新着順」「人気順」で切り替え
- **タグフィルタ**: 左サイドバーのタグをクリックして絞り込み。再クリックで解除
- **AI 検索への導線**: ページ下部のバナーから `/search` へ移動

---

### 記事詳細（`/articles/[id]`）

- 本文は `bodyFormatted`（AI 整形版）があればそちらを表示。ヘッダーに「AI が整形した版を表示中」バッジが出る
- **閲覧数**: ページを開くたびに自動インクリメント（`status: published` のみ）
- **リアクション**: 「いいね」「役に立った」をトグル操作
- **編集・削除ボタン**: 記事の著者本人にのみ表示。他ユーザーには表示されない
- **AI Q&A 導線**: ページ下部から検索ページへ直接移動可能

---

### 記事を書く（`/new`）

#### 推奨フロー（Done is better than perfect）

メモ・下書きをそのまま貼り付けて AI に整えてもらい、数分で公開まで持っていくことを想定しています。

1. タイトルを入力（後から変えてもOK）
2. ドキュメント欄にメモ書き・下書きをそのまま入力
3. **「AI整形」** ボタン → 結果を確認 → 問題なければ「AI整形版を採用する」
4. **「AIタグ提案」** ボタン → 使えるタグをクリックして追加
5. 「公開する」

整形結果に完全に満足できなくても、下書きに留め続けるより公開することを優先してください。その際は以下の運用タグを活用します。

| タグ | 意味 | 使いどころ |
|------|------|---------|
| `WIP` | 作業中・未完成 | まだ書き途中だが役立つ情報がある |
| `古い情報かも` | 内容が古い可能性あり | 正確性に自信がないが削除するには惜しい |
| `要確認` | 誰かに確認してほしい | 事実関係を確かめてから更新したい |

これらのタグは読み手への「免責」ではなく、「不完全でも公開する」という文化の合意として機能します。タグが付いた記事を見た人が加筆・修正する文化につながることを期待しています。

#### AI 整形

- 「AI整形」ボタンをクリックすると、本文が社内文書スタイルに整形されてパネルに表示される
- 「AI整形版を採用する」で保存時に整形版が使われる（原文 `bodyRaw` は必ず保持）
- 整形版を採用しなくてもそのまま原文で公開できる

#### AIタグ提案

- 「AIタグ提案」ボタンをクリックすると、本文をもとにタグ候補が表示される
- チップをクリックして追加 / 再クリックで解除
- 手動でのタグ入力と併用可能

#### AI 編集指示

1. 「AI編集指示」欄に指示を入力（例: 「箇条書きに整理して」）
2. 「依頼」ボタンまたは `Cmd/Ctrl + Enter` で送信
3. 差分ビューが表示される → 「採用する」でエディタに反映 / 「却下」で破棄

#### 下書き自動保存

- 入力中は自動的に localStorage に保存される
- 次回アクセス時に「前回の未保存の内容を復元しました」バナーが表示される
- 「破棄する」で消去可能

---

### 記事編集（`/articles/[id]/edit`）

投稿ページと同じ AI 機能が使用可能。著者本人以外がアクセスした場合は「権限がありません」メッセージを表示。

---

### AI 検索（`/search`）

#### AI 検索モード（デフォルト）

1. 質問を入力（例: 「VPN に接続できない場合どうすればいいですか？」）
2. 「検索」ボタンをクリック
3. AI が関連記事を参照して回答を生成
4. 回答下部の参照記事リンクから原文を確認可能
5. 「役に立ちましたか？」フィードバックボタンで評価を送信

#### 通常検索モード

- モード切り替えトグルで「通常検索」を選択
- タイトル・本文にキーワードが含まれる記事を全文検索
- 一致箇所がハイライト表示される

#### 注意事項

- AI 検索は AI API を呼び出すため数秒かかる
- レート制限あり: 1 分間に 20 回まで（超過時はエラーメッセージに待機秒数を表示）
- 埋め込みベクトルがない記事（シード後に投稿された記事で再インデックスされていないもの）は AI 検索のヒット対象外

---

### 下書き一覧（`/drafts`）

- 選択中のユーザーの下書き記事のみ表示（他ユーザーの下書きは見えない）
- 「編集」ボタンから直接編集ページへ移動

## AI 機能の詳細

### AI プロバイダー切り替え

`.env` の `AI_PROVIDER` で切り替えます。

| 値 | 使用 API | 特徴 |
|----|----------|------|
| `mock` | なし | キーなしで動作。整形は簡易変換、検索はキーワードマッチ |
| `openai` | GPT-4o | 高精度な整形・回答生成 |
| `anthropic` | Claude | `ANTHROPIC_MODEL` で使用モデルを指定可能 |

### 埋め込みと検索の仕組み

全プロバイダーで共通の TF-IDF + 文字バイグラム（256 次元）を埋め込みに使用します。記事投稿・更新時に自動生成され、AI 検索時はコサイン類似度で上位 5 件を取得してコンテキストに使用します。

### レート制限

| エンドポイント | 上限 |
|--------------|------|
| `POST /api/ai/format` | 10 回 / 分 / IP |
| `POST /api/ai/tags` | 10 回 / 分 / IP |
| `POST /api/ai/edit` | 10 回 / 分 / IP |
| `POST /api/search` (AI モード) | 20 回 / 分 / IP |

## 認可モデル（MVP）

| 操作 | 条件 |
|------|------|
| 記事一覧・詳細・検索 | 誰でも閲覧可能 |
| 記事投稿 | ユーザー選択が必要（`X-User-Id` ヘッダー） |
| 記事編集・削除 | 著者本人のみ（サーバー側でヘッダーと `authorId` を照合） |
| 下書き一覧 | 著者本人のみ（`X-User-Id` ヘッダー必須） |
| リアクション | ユーザー選択が必要 |

> MVP では JWT 認証の代わりに `X-User-Id` ヘッダーを使用しています。本番移行時は NextAuth 等の認証基盤に差し替えてください。

---

## 既知の制限

| 制限 | 詳細 |
|------|------|
| 認証なし | `X-User-Id` はブラウザ側で設定するため、技術的にはなりすまし可能。社内 MVP での割り切り |
| 閲覧数のボット計測 | クローラーや連続リロードでも閲覧数が増加する |
| インメモリ Rate Limiter | プロセス再起動でリセット。マルチインスタンス環境では Redis への移行が必要 |
| SQLite | シングルライター。本番では PostgreSQL + pgvector への移行を推奨 |
